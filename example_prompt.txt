Design a comprehensive University Course Registration database. The system must include Students, Departments, Professors, Courses, and a bridge table called Enrollments. A Course belongs to a Department and is taught by exactly one Professor. An Enrollment connects a Student to a Course. Crucial requirements: The Enrollments table must have a composite primary key (student_id, course_id) so a student cannot enroll in the same course twice. Also, add a "grade" column in Enrollments with a CHECK constraint enforcing it to be between 0 and 100. Provide clear relationships.
